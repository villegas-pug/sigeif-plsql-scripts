# Build Excel Unpivot Template

## Gate obligatorio

Antes de ejecutar este flujo, se debe contar obligatoriamente con:

- un `input_file` `.xlsx` generado desde `build-excel-pivot-template`
- un `output_file` `.xlsx` con ruta completa

Si falta cualquiera de esos dos datos, se debe detener la ejecucion y
preguntarlos. No se debe asumir ni inventar valores.

## Que hace

Transforma un archivo Excel pivotado con preguntas horizontales en un archivo
plano listo para carga posterior por otro proceso.

Este flujo:

- conserva `AR_FECHA_REGISTRA`
- conserva `SF_ID_FASE`
- resuelve `PF_ID_FAMILIA` desde un maestro `COD_FAMILIA | PF_ID_FAMILIA`
- despivota cada columna de pregunta a `AP_ID_PREGUNTA`
- mueve el valor de cada celda a `AR_RESPUESTA`
- agrega `AR_USU_REGISTRA = 1`
- no incluye `COD_FAMILIA` en el archivo final

## Cuando usarla

Usar SOLO cuando el usuario pida cualquiera de estos escenarios:

- unpivot de un Excel generado por `build-excel-pivot-template`
- convertir preguntas pivotadas a filas
- preparar un archivo plano para futura insercion en BD por otro proceso
- reconstruir `AP_ID_PREGUNTA` y `AR_RESPUESTA` desde una plantilla SIGEIF

No usar para:

- generar la plantilla pivotada original
- insertar datos directamente en Oracle
- transformar archivos que no tengan la estructura del template SIGEIF

## Estructura esperada del archivo origen

- Fila 1:
  - columnas 1 a 4 vacias
  - desde la columna 5: `AP_PREGUNTA`
- Fila 2:
  - `N° | COD_FAMILIA | AR_FECHA_REGISTRA | SF_ID_FASE`
  - desde la columna 5: `AP_ID_PREGUNTA`
- Fila 3 en adelante:
  - datos digitados manualmente

## Maestro de familias

El script busca por defecto un archivo llamado:

`cod_familia_+_id_Familia.xlsx`

en la misma carpeta del `input_file`.

Ese archivo debe contener exactamente estas columnas:

- `COD_FAMILIA`
- `PF_ID_FAMILIA`

## Estructura final esperada

El Excel de salida debe tener estas columnas en este orden:

- `PF_ID_FAMILIA`
- `AR_FECHA_REGISTRA`
- `SF_ID_FASE`
- `AP_ID_PREGUNTA`
- `AR_RESPUESTA`
- `AR_USU_REGISTRA`

## Flujo operativo

1. Verificar que exista `input_file`.
2. Verificar que exista `output_file` y que termine en `.xlsx`.
3. Leer el archivo pivotado respetando fila 1 y fila 2 como metadatos.
4. Resolver el maestro `COD_FAMILIA | PF_ID_FAMILIA` en la misma carpeta del input.
5. Validar que el maestro contenga las columnas requeridas.
6. Despivotar las columnas de preguntas a filas.
7. Mantener `AR_RESPUESTA` vacia en blanco.
8. Exportar el resultado sin la columna `COD_FAMILIA`.
